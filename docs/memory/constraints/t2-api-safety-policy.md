# Memory: constraints/t2-api-safety-policy
Updated: now

## 외부 유료 API 호출 시 절대 원칙 — 위반 시 과금 사고 발생

### 1. while(true) 루프로 유료 API를 반복 호출하지 않음
- 오케스트레이터 함수에서 하위 API 함수를 루프로 연속 호출 금지
- 한 번의 cron 실행 = 한 번의 배치 처리, 다음 cron에서 이어서 진행
- offset/진행 상태는 반드시 DB에 기록하여 중복 방지

### 2. 하드 리밋(Hard Limit)은 코드에 반드시 포함
- 유료 API를 호출하는 모든 함수에 일일 최대 호출 수를 하드코딩
- 예산 추정이 아닌 실제 호출 로그 기반으로 카운팅
- 리밋 도달 시 즉시 중단 + 로그 기록 (절대 "추정"으로 넘기지 않음)

### 3. 실행 잠금(Execution Lock)
- 같은 함수가 동시에 여러 번 실행되지 않도록 DB 기반 잠금 적용
- 잠금 시간 초과(TTL) 설정으로 데드락 방지

### 4. 과금 구조를 코드에 반영
- API 플랜(BASIC/PRO/ULTRA)의 실제 한도를 코드 상수에 정확히 반영
- 플랜 변경 시 코드 상수도 반드시 업데이트
- "Pro plan daily limit"이라고 주석만 달고 실제로는 BASIC인 상황 금지

### 5. 빈 응답도 과금됨을 인지
- API가 빈 응답({}, [])을 반환해도 호출 카운트는 올라감
- 빈 응답이 연속 N회 발생 시 해당 배치 중단 로직 필수
- 데이터 없이 과금만 되는 상황을 감지하는 모니터링 필수

### 6. 신규 유료 API 연동 시 체크리스트
- [ ] 구독 플랜 및 한도 확인
- [ ] 일일 하드 리밋 상수 설정
- [ ] 실제 API 호출 수 로깅
- [ ] 중복 실행 방지 잠금
- [ ] 빈 응답 연속 감지 중단 로직
- [ ] while 루프 사용 여부 점검 (금지)
